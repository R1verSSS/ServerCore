# Роли доступа

ServerCore использует централизованную модель доступа. Проверка должна работать не только на уровне Discord slash-команд, но и для кнопок, select-menu, старых опубликованных панелей и web-действий.

## Уровни доступа

| Уровень | Назначение | Примеры возможностей |
|---|---|---|
| Member | обычный участник | профиль, daily, магазин, тикеты, роли, темы |
| VIP | расширенный участник | дополнительные пользовательские возможности, бонусы магазина/профиля |
| Helper | помощник | просмотр тикетов, заявок, предупреждений, помощь пользователям |
| Moderator | модератор | warn, mute, clear, cases, moderation history |
| Admin | администратор | настройки, панели, AutoMod, backup/export, events, tournaments |
| Owner | владелец | maintenance, restore, критичные hosting/system actions |

## Рекомендуемые Discord-роли

```text
👑 Owner
🛡 Admin
👮 Moderator
🧰 Helper
💎 VIP
```

## Принципы безопасности

- Discord permissions скрывают часть команд, но не являются единственной защитой.
- Серверная проверка доступа обязательна внутри бота.
- Старые опубликованные кнопки тоже должны проверять права при нажатии.
- При отказе лучше показывать понятное сообщение: какая роль нужна и почему действие недоступно.

## Примеры разграничения

| Действие | Member | Helper | Moderator | Admin | Owner |
|---|---:|---:|---:|---:|---:|
| `/me`, `/profile`, `/daily` | ✅ | ✅ | ✅ | ✅ | ✅ |
| Создать тикет | ✅ | ✅ | ✅ | ✅ | ✅ |
| Смотреть пользовательские заявки | ❌ | ✅ | ✅ | ✅ | ✅ |
| Warn / mute / clear | ❌ | ❌ | ✅ | ✅ | ✅ |
| AutoMod settings | ❌ | ❌ | ❌ | ✅ | ✅ |
| Backup / export | ❌ | ❌ | ❌ | ✅ | ✅ |
| Maintenance / restore | ❌ | ❌ | ❌ | ❌ | ✅ |
