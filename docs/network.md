# Диагностика сети Discord

Если бот получает slash-команду, но не может ответить, а в консоли есть `UND_ERR_CONNECT_TIMEOUT`, это обычно проблема маршрута Node.js до Discord REST API.

Проверь локально:

```bash
npm run net:check
npm run doctor
```

Также можно открыть веб-панель:

```text
http://localhost:3000/network
```

Что помогает чаще всего:

1. Системный VPN, а не расширение браузера.
2. Разрешить `node.exe` в Windows Firewall.
3. Использовать Node.js LTS 20/22 вместо экспериментальной версии.
4. Запустить бота на VPS/Railway/Render.

Проверка через браузер не всегда показательна: браузер может идти через VPN-расширение, а Node.js — напрямую.
