# v24.1.30 Ticket Flow Upgrade

Обновлена логика тикетов поддержки.

## Новая схема

- `📨・заявки` — только заявки/анкеты и forum-посты заявок.
- Активные тикеты создаются как приватные текстовые каналы в отдельной категории `🎫 ПОДДЕРЖКА`.
- Пользователь и поддержка общаются прямо в созданном канале `ticket-username`.
- После закрытия тикета канал удаляется, а итоговая карточка сохраняется в архив `📁・тикеты`.

## Переменные окружения

```env
TICKET_ACTIVE_CATEGORY_NAME=🎫 ПОДДЕРЖКА
TICKET_ACTIVE_CATEGORY_ID=
TICKET_ARCHIVE_CHANNEL_NAME=📁・тикеты
TICKET_ARCHIVE_CHANNEL_ID=
```

Если указан ID, он имеет приоритет над именем.

## Права бота

Для активных тикетов нужны права:

- View Channels
- Manage Channels
- Send Messages
- Read Message History

Для архива закрытых тикетов нужны права отправки сообщений в `📁・тикеты`.
