# v24.1 User Experience Flow Upgrade

Обновление делает пользовательский путь более понятным: участнику не нужно помнить команды, бот сам показывает следующий логичный шаг.

## Что добавлено

1. **Умный “Мой центр”**
   - `/me` теперь открывает Smart User Center.
   - Показывает уровень, XP, монеты, Daily cooldown, тикеты, темы, заявки и бейджи.
   - Кнопки Daily / Ticket / Shop меняют состояние по данным пользователя.

2. **Динамические действия Daily / Ticket / Shop**
   - Кнопка Daily отключается, если награда на cooldown.
   - Тикет показывает, есть ли уже открытый тикет.
   - После действий появляется блок “Что дальше?”.

3. **Действия из пользовательской веб-панели**
   - В `/user-panel` добавлен блок “Что дальше?”.
   - Пользователь может получить web-Daily и перейти к своим тикетам/темам.
   - Все web-действия пишутся в `webUserActions` и `userActionLog`.

4. **Мои темы / мои тикеты / мои заявки**
   - В Discord: кнопка `Мои списки` в `/me`.
   - В веб-панели: `/my-topics`, `/my-tickets`, `/my-applications`.
   - Forum-темы теперь сохраняются в базе в `forumThreads`.

5. **Onboarding с наградой и бейджем**
   - За завершение чек-листа начисляются монеты.
   - Дополнительно выдается достижение `onboarding_complete` и бейдж 🧭.

6. **Шаблоны создания forum-тем**
   - Существующая forum-панель теперь записывает созданные темы в пользовательскую историю.
   - После создания темы показывается подсказка, что делать дальше.

7. **Подсказки “что дальше?” после действий**
   - После Daily, тикета и создания темы бот предлагает следующий шаг.

8. **Расширенные достижения**
   - Добавлены достижения:
     - `onboarding_complete` — завершить onboarding.
     - `first_forum_topic` — создать первую forum-тему.
     - `first_role_select` — выбрать роль через select-menu.

9. **Выбор ролей через select-menu**
   - `/roles` теперь отправляет select-menu вместо набора кнопок.
   - Можно выбрать несколько интересов за одно действие.
   - Повторный выбор роли снимает ее.

10. **Ограничения кнопок по ролям**
   - Существующий `accessControlService` продолжает проверять admin/moderator-кнопки.
   - Пользовательские кнопки `ux:*`, `role:*`, `menu:quick:*` оставлены доступными для участников.

## Проверка

```bash
npm run check
npm run deploy
npm run setup
```

После setup переопубликуй панели:

- `/roles` — новая select-menu панель ролей.
- `/me` — проверить Smart User Center.
- `/threadpanel post` — проверить создание forum-тем и раздел “Мои темы”.
